# Phase 4: Dashboard, Results UI, and Real Automation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the results dashboard, implement real Playwright browser automation, add test status tracking, and wire up Slack notifications.

**Architecture:** 
- Frontend: Dashboard (`/dashboard`) + results pages (`/dashboard/[siteId]`) with components for test plan, audit results, and metrics visualization
- Backend: Real Playwright audit implementation using `@playwright/test`, test status tracking in Supabase, async analysis jobs, Slack webhook integration
- Database: New `test_runs` table to track analysis status (pending → running → completed); migrate `test_plans` and `audit_results` to reference test runs

**Tech Stack:** Next.js 15, React 19, Playwright (headless), Supabase, Slack API, TailwindCSS

## Global Constraints

- Minimum code; no speculative features per coding-discipline.md
- Playwright audit must run in headless mode (production-ready)
- All Slack notifications are best-effort (non-blocking if fail)
- No error handling for impossible scenarios; trust framework guarantees
- Test tracking status values: `pending` → `running` → `completed` → `error`
- Database RLS unchanged; all queries use service_role bypasses

## Task 1: Database Schema - Test Run Tracking

**Files:**
- Create: `supabase/migrations/004_test_runs_table.sql`

**Interfaces:**
- Produces: `TestRun` type with fields: `id` (uuid), `site_id` (uuid), `status` (enum: pending/running/completed/error), `started_at` (timestamp), `completed_at` (timestamp), `error_message` (text, nullable), `results` (jsonb, nullable)

### Steps

- [ ] **Step 1: Write the migration**

```sql
-- Create test_runs table to track analysis jobs
create table test_runs (
  id uuid default gen_random_uuid() primary key,
  site_id uuid not null references sites(id) on delete cascade,
  status text not null check (status in ('pending', 'running', 'completed', 'error')),
  started_at timestamp with time zone default now(),
  completed_at timestamp with time zone,
  error_message text,
  results jsonb,
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now()
);

-- Add indexes for queries
create index idx_test_runs_site_id on test_runs(site_id);
create index idx_test_runs_status on test_runs(status);
create index idx_test_runs_created_at on test_runs(created_at desc);

-- Update test_plans and audit_results to reference test_run_id
alter table test_plans add column test_run_id uuid references test_runs(id) on delete cascade;
alter table audit_results add column test_run_id uuid references test_runs(id) on delete cascade;

-- Enable RLS
alter table test_runs enable row level security;

-- RLS policy: allow all access via service_role (app uses service_role)
create policy "allow_service_role" on test_runs
  as permissive for all to service_role
  using (true) with check (true);
```
